# Домашнее задание 3

## Вариант

Вариант 9: справочник `Страны`, основная таблица `Города`, числовое поле `population_k` - население в тысячах человек.

---

## Архитектура и Технологический стек

Проект переведен со старого консольного стека (ADO.NET + ручной SQL) на современный веб-стек .NET 8.0/10.0:
- **Язык программирования**: C#
- **База данных**: SQLite
- **Доступ к данным**: Entity Framework Core (Code First) — без прямых SQL-запросов.
- **Запросы к данным**: LINQ to Entities.
- **Интерфейс пользователя**: ASP.NET Core MVC (модель-представление-контроллер) с адаптивной версткой и оформлением.

---

## Реализованная структура проекта (`DZ3_ASOIU/`)

1. **Модели данных (`Models/`)**:
   - `Country.cs` — сущность страны (справочник). Содержит навигационное свойство `ICollection<City> Cities`.
   - `City.cs` — сущность города. Содержит навигационную ссылку `Country? Country` и валидацию `PopulationK >= 0` на уровне свойств и атрибутов.
   - `AppDbContext.cs` — контекст EF Core. Настроен на SQLite (`cities_ef.db`).
     - Метод `InitializeDatabase()` автоматически выполняет `EnsureCreated()`.
     - При пустой БД автоматически считывает исходные файлы `country.csv` и `city.csv`, сохраняя оригинальные связи между ключами.
   - `ReportsViewModel.cs` — специальная DTO/ViewModel для передачи результатов тройного аналитического отчета в представление.

2. **Слой представления и логики контроллеров (`Controllers/` и `Views/`)**:
   - `CountriesController.cs` & `Views/Countries/` — полный CRUD для справочника стран.
     - **Важная особенность**: При попытке удалить страну, в которой еще есть города, система на уровне UI выводит предупреждение и полностью блокирует удаление (согласно требованию ТЗ).
   - `CitiesController.cs` & `Views/Cities/` — полный CRUD для таблицы городов.
     - Добавление и редактирование городов выполняются через выпадающий список (загружается из базы через LINQ-запрос к странам).
     - Поля валидируются на стороне клиента и сервера.
   - `ReportsController.cs` & `Views/Reports/` — аналитический отчет.
     - Все три отчета объединены на одной странице отчетов (с переключением вкладок):
       - **Раздел 1**: Полный список городов с названием стран (`Include(c => c.Country).OrderBy(c => c.Name)`).
       - **Раздел 2**: Количество городов по странам (`GroupBy(c => c.Country!.Name).Select(...)`).
       - **Раздел 3**: Среднее население по странам, отсортированное по убыванию среднего (`GroupBy(c => c.Country!.Name).Average(...)`).
   - `Views/Shared/_Layout.cshtml` — общий макет с навигационной панелью, адаптированной под все устройства.

3. **Стилевое оформление (`web/css/site.css`)**:
   - Реализована темная тема оформления с адаптивными таблицами и формами.
   - Стили для карточек, кнопок при наведении на кнопки и строки таблиц.
   - Бэджи для стран и численности населения.

## Сборка и Запуск

Проект успешно собирается и готов к запуску. В корне решения `DZ3_ASOIU` находятся необходимые CSV-файлы.

### Шаг 1. Сборка проекта
```bash
dotnet build DZ3_ASOIU.sln
```

### Шаг 2. Запуск веб-приложения
```bash
dotnet run --project DZ3_ASOIU/DZ3_ASOIU.csproj
```

После запуска приложение будет доступно в браузере по адресу, указанному в консоли (обычно `http://localhost:5000` или `https://localhost:5001`).
