# Проект «Страны и города» — Домашнее задание 3

Проект представляет собой веб-приложение на платформе **ASP.NET Core MVC** (.NET 8.0/10.0), предназначенное для администрирования справочника стран и основной таблицы городов с их показателями населения.

Работа с данными выполнена через: **Entity Framework Core (Code First)**, **LINQ to Entities** и СУБД **SQLite**.

---

## Почему именно веб-интерфейс (ASP.NET Core MVC)?

В соответствии с требованиями Домашнего задания №3, вместо текстового консольного меню необходимо было разработать графический или веб-интерфейс. Выбран веб-интерфейс на базе **ASP.NET Core MVC** по следующим причинам:

### 1. Нативная кроссплатформенность (Cross-platform)
Традиционные настольные графические интерфейсы на C# (такие как **Windows Forms** или **WPF**) жестко привязаны к операционной системе Windows. Поскольку разработка и запуск производятся в среде **Linux**, использование WinForms или WPF привело бы к критической несовместимости или необходимости использовать сложные и нестабильные эмуляторы (например, Mono/Wine).
**ASP.NET Core** работает нативно, быстро и стабильно на **Linux, macOS и Windows**, обеспечивая переносимость решения.

### 2. Пользовательский интерфейс
Использование связки HTML5, CSS3 с кастомными стилями и JS позволяет создать современный, живой и визуально привлекательный интерфейс с адаптивной версткой. Пользователю не требуется устанавливать шрифты или настраивать графическую подсистему — браузер отображает интерфейс идеально на любом устройстве (от смартфона до широкоформатного монитора).

### 3. Отсутствие затрат на развертывание (Zero Install)
Для работы с классическим настольным приложением клиент должен скачивать исполняемые файлы, библиотеки и устанавливать их. Веб-интерфейс требует развертывания только на сервере. Любой клиент может получить мгновенный доступ к системе, просто введя URL-адрес в браузере.

### 4. Четкое разделение ответственности (MVC-паттерн)
Паттерн Model-View-Controller идеально структурирует кодовую базу:
* **Model (Модель)**: Классы данных `Country` и `City` с декларативной валидацией.
* **View (Представление)**: Чистые и легкие Razor-шаблоны без логики БД.
* **Controller (Контроллер)**: Тонкие контроллеры, управляющие маршрутизацией и передачей данных.

## Технологический стек

* **Язык**: C# 12 / .NET 8.0/10.0
* **База данных**: SQLite
* **ORM**: Entity Framework Core 10 (Code First)
* **Запросы к данным**: LINQ to Entities (строгая типизация запросов, отсутствие сырых SQL-строк)
* **Архитектурный шаблон**: ASP.NET Core MVC
* **Дизайн**: Кастомный CSS (Glassmorphism, темная тема, плавные анимации, отзывчивые таблицы) + Bootstrap 5.

## Основные возможности

1. **Справочник стран (Master)**:
   * Просмотр списка стран.
   * Добавление новых стран и редактирование существующих.
   * **Безопасное удаление**: Система проверяет наличие городов в стране и блокирует удаление на уровне UI и сервера, если в стране есть привязанные города (с выводом предупреждения).
2. **Список городов (Detail)**:
   * Просмотр с автоподгрузкой названия страны (через `Include`).
   * Добавление/редактирование с удобным выпадающим списком стран, формируемым через LINQ.
   * Встроенная валидация населения на клиенте и сервере (значение $\ge 0$).
3. **Аналитический отчет (LINQ Dashboard)**:
   * Все три отчета объединены на одной вкладке и формируются исключительно средствами LINQ:
     * *Раздел 1*: Полный список городов с названием стран (`Include` + `OrderBy`).
     * *Раздел 2*: Количество городов по каждой стране (`GroupBy` + `Count`).
     * *Раздел 3*: Среднее население городов по странам, отсортированное по убыванию (`GroupBy` + `Average` + `OrderByDescending`).
4. **Автоматический сидинг (Seeding)**:
   * При первом старте приложение автоматически создает файл базы данных `cities_ef.db` через `EnsureCreated()`.
   * Если таблицы пусты, они автоматически наполняются из оригинальных файлов `country.csv` и `city.csv`, сохраняя оригинальные ID и связи.

## Инструкция по запуску

Проект полностью настроен и готов к сборке.

### 1. Сборка проекта
```bash
dotnet build DZ3_ASOIU.sln
```

### 2. Запуск приложения
```bash
dotnet run --project DZ3_ASOIU/DZ3_ASOIU.csproj
```

После этого откройте браузер и перейдите по адресу, который отобразится в консоли:
* **`http://localhost:5115`**

## Структура файлов проекта

* `DZ3_ASOIU/Models/` — Entity-модели (`Country.cs`, `City.cs`), контекст `AppDbContext.cs` и ViewModel отчетов.
* `DZ3_ASOIU/Controllers/` — Контроллеры для стран, городов и отчетов.
* `DZ3_ASOIU/Views/` — Razor-представления для каждого из разделов и общая мастер-страница `_Layout.cshtml`.
* `DZ3_ASOIU/web/` — Статические ресурсы, включая кастомную таблицу стилей `css/site.css` с темной темой.
* `DZ3_ASOIU/country.csv` и `city.csv` — CSV-файлы с тестовыми данными.
